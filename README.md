# Full-Stack React Project (JSON Server)

פרויקט זה פותח כחלק ממטלת בניית יישום React בגישת Full-Stack תוך שימוש בשרת Mock מבוסס `json-server`. 
נכון לעכשיו, הפרויקט כולל את שני השלבים הראשונים של המטלה (חלק ב' וחלק ג').

---

## 📂 מבנה הפרויקט

הפרויקט מחולק לשני חלקים מרכזיים:
1. **צד שרת (Backend):** ממוקם בתיקייה הראשית. מבוסס על `json-server` ומכיל את מסד הנתונים `db.json` ותיקיית `public/images/` עם תמונות מקומיות.
2. **צד לקוח (Frontend):** ממוקם בתוך תיקיית `client/`. מבוסס על `Vite` ו-`React`.

---

## 📝 תיאור השלבים שבוצעו

### חלק ב' – שרת הנתונים (JSON-Server)
* **משיכת נתונים:** נמשכו נתונים מהאתר `jsonplaceholder.typicode.com` עבור משתמשים (Users), פוסטים (Posts), אלבומים (Albums), תמונות (Photos) ומשימות (Todos). המידע נשמר באופן מאורגן בקובץ `db.json`.
* **תמונות לוקאליות:** הורדו 20 תמונות איכותיות (בנושא אוכל) אל תוך התיקייה `public/images/`. קובץ ה-`db.json` עודכן כך שכל התמונות באלבומים מצביעות אל התמונות המקומיות האלו במקום לכתובות חיצוניות (לדוגמה: `/images/food3.jpg`).
* **הגדרת השרת:** הוגדרה פקודת `start` בקובץ `package.json` כדי להפעיל את השרת על פורט `3000`.

### חלק ג' – יישום React
נבנתה תשתית צד הלקוח בעזרת Vite, תוך הקפדה על כלים שנלמדו בקורס (Function Components, Hooks, React Router v6, Axios):
* **תשתית וניתוב:** הוגדר `BrowserRouter` מתוך `react-router-dom` לניהול המסכים השונים.
* **עמוד התחברות (`/login`):** מאפשר כניסה למערכת. מבצע אימות מול השרת (שם משתמש תואם לשדה `username`, סיסמה תואמת לשדה `website`).
* **עמוד הרשמה (`/register`):** תהליך הרשמה מפוצל ל-2 שלבים. מאמת את זמינות שם המשתמש, קולט נתונים ורושם את המשתמש במסד הנתונים באמצעות בקשת `POST`.
* **עמוד הבית (`/users/:userId/home`):** לוח בקרה מרכזי (Dashboard). כולל סרגל ניווט (Navbar) מתקדם, כפתור להתנתקות וניקוי ה-`localStorage`, וכפתור `Info` הפותח חלון Modal מעוצב להצגת פרטי המשתמש.
* **עיצוב (Styling):** הפרויקט מעוצב בגישת Glassmorphism מודרנית ו-Dark Mode, כפי שמוגדר בקובץ `client/src/index.css`.

## 🚀 הוראות הרצה (איך להפעיל את הפרויקט)

כדי שהאפליקציה תעבוד באופן מלא, **יש להריץ במקביל גם את השרת וגם את צד הלקוח** בשני חלונות טרמינל נפרדים.

### צעד 1: הפעלת השרת (Backend)
1. פתחו טרמינל בתיקיית השורש של הפרויקט (איפה שקובץ ה-`package.json` הראשי נמצא).
2. הריצו את הפקודה:
   ```bash
   npm start
   ```
3. השרת יעלה ויאזין לבקשות בכתובת: `http://localhost:3000`

### צעד 2: הפעלת צד הלקוח (Frontend)
1. פתחו חלון טרמינל **נוסף**.
2. נווטו אל תוך תיקיית הלקוח:
   ```bash
   cd client
   ```
3. הריצו את הפקודה:
   ```bash
   npm run dev
   ```
4. האפליקציה תעלה ותהיה זמינה בדפדפן, לרוב בכתובת: `http://localhost:5173`
